# 판타지 세계 속 나의 캐릭터

> 상황형 질문을 통해, 판타지 세계 속에서의 역할과 생존 방식을 캐릭터 리포트 형태로 보여주는 Streamlit 프로젝트

이 프로젝트는 사용자의 선택을 바탕으로  
**세계관**, **역할군**, **보조 태그**, **프롤로그**, **진행 조언**을 조합해  
하나의 캐릭터 리포트를 생성하는 Streamlit 웹 애플리케이션입니다.

중간고사 대체 과제를 위해 제작했으며,  
과제 필수 요구사항인 **로그인 기능**, **캐싱 기능**, **퀴즈 기능**을 포함하고 있습니다.

- 학부: 정보융합학부

## 1. 프로젝트 소개

이번 과제의 주제가 자유주제였기 때문에,  
단순한 상식 퀴즈나 일반적인 성격검사보다 **조금 더 서사적인 형태의 앱**을 만들고 싶었습니다.

그래서 이 프로젝트는  
“판타지 세계에 떨어졌을 때 나는 어떤 방식으로 움직이고, 어떤 역할로 살아남게 될까?”  
라는 질문에서 시작했습니다.

보통 성향 테스트는 결과가 짧게 끝나는 경우가 많은데,  
저는 결과를 단순히 한 줄짜리 유형으로 보여주기보다  
**하나의 캐릭터 해석 리포트처럼 보이게 만드는 것**을 목표로 했습니다.

그래서 사용자의 응답을 바탕으로 다음 요소를 함께 조합하도록 설계했습니다.

- 세계관
- 역할군
- 태그 2개
- 프롤로그
- 진행 조언

즉, 이 프로젝트는 단순한 성격검사라기보다  
**“상황형 선택을 통해 하나의 판타지 캐릭터를 구성해 가는 앱”**에 가깝습니다.

## 2. 기획 의도

이 앱에서 가장 중요하게 생각한 부분은 **몰입감**이었습니다.

같은 질문이라도  
“나는 계획적인 편이다”  
같은 직설적인 문장보다

- 낯선 세계에 처음 떨어졌을 때 무엇을 먼저 확인하는지
- 적이나 위협 앞에서 어떤 방식으로 움직이는지
- 동료와 조직 안에서 어떤 역할을 맡게 되는지
- 오래 버티는 사람의 방식은 무엇인지

처럼 **상황 안에서 자연스럽게 선택하게 만드는 방식**이 더 재미있고,  
결과도 캐릭터처럼 읽히기 좋다고 생각했습니다.

그래서 질문 전체를  
**설명형 문항이 아니라 세계관형 상황 질문** 중심으로 구성했습니다.

## 3. 주요 기능

### 로그인 기능
- 아이디와 비밀번호를 입력해 로그인할 수 있습니다.
- 로그인 성공/실패에 따라 다른 메시지를 출력합니다.
- 로그인 상태는 `st.session_state`를 통해 유지됩니다.

### 캐싱 기능
- 질문 데이터, 사용자 데이터, 세계관 데이터, 역할군 데이터, 결과 텍스트 데이터를 불러오는 함수에 `st.cache_data`를 적용했습니다.
- 같은 JSON 파일을 반복해서 읽지 않도록 하여 앱 실행 효율을 높였습니다.

### 퀴즈 기능
- 총 **60문항**의 상황형 질문으로 구성되어 있습니다.
- 각 문항은 **4지선다형**입니다.
- 사용자는 가장 끌리는 선택지를 하나 고르게 됩니다.
- 선택 결과에 따라 점수를 누적하고, 최종 캐릭터 리포트를 생성합니다.

### 랜덤 선택 결과 보기
- 사용자가 직접 60문항을 모두 풀지 않아도 결과를 빠르게 볼 수 있도록 구현했습니다.
- 단순히 결과를 랜덤으로 뽑는 방식이 아니라,  
  **60문항 전체의 선택지를 랜덤으로 자동 응답한 뒤 실제 점수 계산 로직을 그대로 적용**하도록 만들었습니다.

### 결과 리포트 기능
결과 화면에서는 아래 내용을 확인할 수 있습니다.

- 선택된 세계관
- 대표 역할군
- 세계관별 호칭
- 보조 태그 2개
- 프롤로그
- 진행 조언
- 결과 다운로드

### 캐릭터 아카이브 기능
- 사용자가 세계관, 역할군, 태그를 직접 선택해 결과를 미리 볼 수 있도록 구성했습니다.
- 실제 검사 결과가 아니라, 조합별 분위기를 확인하는 프리뷰 기능입니다.

### 결과 시각화 기능
- 세계관 점수
- 역할군 점수
- 상위 태그 점수
- 빅파이브 보정 점수(참고용)

결과 화면에서는 Plotly를 이용해 주요 점수를 시각적으로 확인할 수 있습니다.

## 4. 결과 구성과 계산 방식

이 프로젝트의 결과는 아래 요소를 조합해 생성됩니다.

- **세계관 6개**  
  아포칼립스 / 다크 판타지 / 정통 판타지 / 코미디 판타지 / 사이버펑크 / 무협

- **역할군 8개**  
  전위 / 사수 / 술자 / 지원가 / 정찰자 / 기술자 / 협상가 / 지휘관

- **태그 10개**  
  성장형 / 생존형 / 탐구형 / 공략형 / 보조형 / 운영형 / 유랑형 / 재기형 / 계약형 / 기록형

- **결과 서사**  
  프롤로그 / 진행 조언

사용자는 총 **60문항**에 응답하게 되며,  
각 문항의 선택지는 아래 4가지 점수 축에 동시에 연결됩니다.

- 세계관 점수
- 역할군 점수
- 태그 점수
- 빅파이브 보정 점수

즉, 한 개의 선택지를 고를 때도 단일 점수만 오르는 것이 아니라,  
**여러 축의 점수가 함께 누적되도록 설계**했습니다.

예를 들면 특정 선택지는 다음처럼 연결될 수 있습니다.

- 세계관: 사이버펑크 +1
- 역할군: 기술자 +2
- 태그: 공략형 +1, 탐구형 +1
- 빅파이브: 개방성 +1, 성실성 +1

최종 결과는 60문항 전체 점수를 합산한 뒤 아래 순서로 결정됩니다.

1. 세계관 점수 최고값 1개 선택
2. 역할군 점수 최고값 1개 선택
3. 태그 점수 상위 2개 선택
4. 빅파이브 보정 점수를 참고해 결과 해석 보정
5. 선택된 조합을 바탕으로 프롤로그와 진행 조언 생성

이 구조를 통해 결과가 단순히 “어떤 유형인지”에 그치지 않고,  
**이 세계에서 어떤 방식으로 움직이는 인물인지**까지 자연스럽게 이어지도록 만들었습니다.

---

## 5. 문항 설계 방식

문항은 총 6개 섹션으로 나누어 구성했습니다.

1. 낯선 세계에 떨어진 첫 순간  
2. 위협 앞에서 움직이는 방식  
3. 힘과 기술을 다루는 방식  
4. 동료와 조직 속의 나  
5. 오래 버티는 사람의 방식  
6. 끝내 남는 이름과 방식

이렇게 나눈 이유는 단순히 문항 수를 분배하기 위해서가 아니라,  
**결과를 단계적으로 쌓아 가는 구조**를 만들기 위해서입니다.

예를 들어 앞쪽 섹션에서는  
세계관 선호와 첫 반응, 위기 대처 방식 등을 많이 반영하고,

중간 섹션에서는  
직접 전투형인지, 기술형인지, 지원형인지 같은 역할군을 가르고,

후반 섹션에서는  
생존형인지, 기록형인지, 재기형인지처럼 결과의 결을 정리하도록 설계했습니다.

즉, 60문항이 전부 같은 무게로 흩어져 있는 것이 아니라  
**섹션별로 역할을 분담한 구조**라고 볼 수 있습니다.

---

## 6. 결과 표현 방식

결과 화면은 단순 점수표보다 **캐릭터 리포트에 가까운 느낌**으로 보이도록 구성했습니다.

그래서 아래 순서를 기준으로 화면을 설계했습니다.

1. 핵심 결과 요약
2. 세계관 설명
3. 프롤로그
4. 진행 조언
5. 점수 시각화
6. 내부 점수 참고

특히 프롤로그는  
“당신은 ~~입니다”처럼 고정된 문장을 그대로 출력하는 것이 아니라,  
**세계관 + 역할군 + 태그 조합에 따라 문장 조각을 조합하는 방식**으로 만들었습니다.

이 방식 덕분에 결과가 조금 더 서사적으로 읽히고,  
같은 역할군이라도 세계관과 태그가 달라지면 분위기가 자연스럽게 달라지도록 구성할 수 있었습니다.

## 7. 예시 결과

예를 들어 다음과 같은 결과가 나올 수 있습니다.

- 세계관: **사이버펑크**
- 역할군: **기술자**
- 이 세계에서의 이름: **엔지니어**
- 태그: **탐구형 / 공략형**

또는

- 세계관: **무협**
- 역할군: **전위**
- 이 세계에서의 이름: **검객**
- 태그: **성장형 / 생존형**

같은 역할군이라도  
세계관과 태그 조합에 따라 결과의 분위기와 해석이 달라지도록 설계했습니다.

## 10. 데이터 구성

### questions.json

* 총 60개의 문항이 저장되어 있습니다.
* 각 문항은 문항 번호, 섹션 번호, 질문 내용, 4개 선택지, 점수 매핑 정보를 포함합니다.
* 각 선택지는 세계관 / 역할군 / 태그 / 빅파이브 보정 점수에 연결됩니다.

### worlds.json

* 6개의 세계관 정보를 저장합니다.
* 각 세계관의 이름, 한 줄 요약, 상세 설명이 포함됩니다.

### role_titles.json

* 기본 역할군 이름과 세계관별 역할군 호칭을 저장합니다.
* 태그의 한글 이름도 함께 저장합니다.

### result_texts.json

* 프롤로그와 진행 조언 생성을 위한 텍스트 조각을 저장합니다.
* 세계관 문장, 역할군 문장, 태그 문장, 조언 문장, 특수 조합 override 문장을 포함합니다.

### users.json

* 로그인용 사용자 정보를 저장합니다.
* 아이디, 비밀번호, 이름을 포함합니다.

## 11. 캐싱 설명

이 프로젝트에서는 Streamlit의 `st.cache_data`를 활용하여 아래 데이터를 캐싱합니다.

* 질문 데이터 (`questions.json`)
* 사용자 데이터 (`users.json`)
* 세계관 데이터 (`worlds.json`)
* 역할군/태그 데이터 (`role_titles.json`)
* 결과 텍스트 데이터 (`result_texts.json`)

이렇게 캐싱을 적용한 이유는,
앱이 다시 실행되거나 페이지가 새로고침되더라도 같은 JSON 파일을 반복해서 읽지 않도록 하기 위해서입니다.

특히 질문 데이터와 결과 텍스트 데이터는 앱 실행 중 여러 번 참조되므로,
캐싱을 통해 불필요한 파일 입출력을 줄이고 앱 실행을 더 안정적으로 유지할 수 있습니다.

## 12. 앱 사용 흐름

1. 첫 화면에서 학번과 이름을 확인합니다.
2. 로그인 정보를 입력하여 로그인합니다.
3. 메뉴에서 직접 검사 또는 랜덤 선택 결과 보기를 선택합니다.
4. 직접 검사 시 60문항에 응답합니다.
5. 결과 화면에서 세계관, 역할군, 태그, 프롤로그, 진행 조언을 확인합니다.
6. Plotly 그래프로 점수를 확인할 수 있습니다.
7. 필요하면 결과를 텍스트 파일로 다운로드할 수 있습니다.
8. 캐릭터 아카이브에서 원하는 조합의 결과를 직접 미리 볼 수 있습니다.

## 13. 느낀 점

이번 프로젝트에서는 단순히 Streamlit 기능을 붙이는 것보다,
**기획과 데이터 구조 설계가 결과 품질에 얼마나 큰 영향을 주는지**를 많이 느꼈습니다.

특히 60문항을 단순히 나열하는 대신
섹션별 역할을 나누고, 각 선택지가 여러 점수 축에 연결되도록 설계하면서
결과가 훨씬 자연스럽고 다양하게 나오도록 만들 수 있었습니다.

또한 JSON 파일을 분리해서 관리하면서
질문 데이터, 결과 텍스트, 세계관 정보, 역할군 호칭을 독립적으로 다룰 수 있었고,
이런 구조가 이후 수정과 확장에도 훨씬 유리하다는 점을 확인할 수 있었습니다.

기술적으로는 Streamlit의 세션 상태 관리와 캐싱 기능을 실제로 적용해 보면서
단순한 화면 출력 이상의 웹앱 흐름을 경험할 수 있었습니다.
