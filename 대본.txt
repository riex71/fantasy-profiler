1. 시작 화면
화면

앱 첫 화면 보여주기
학번 / 이름이 보이게 잠깐 멈추기

대본

안녕하세요. 중간고사 대체 과제로 제작한 Streamlit 웹 애플리케이션 판타지 세계 속 나의 캐릭터입니다.
이 프로젝트는 사용자의 성향을 바탕으로, 판타지 세계에서 어떤 역할과 태그를 가진 캐릭터인지 리포트 형식으로 보여주는 앱입니다.
첫 화면에는 과제 조건에 맞게 제 학번과 이름이 표시되도록 구성했습니다.

2. 로그인 기능 시연
화면

아이디/비밀번호 입력
일단 한 번 틀리게 넣어도 좋고, 바로 맞게 넣어도 됨

대본

다음으로 로그인 기능입니다.
사용자는 아이디와 비밀번호를 입력해서 로그인할 수 있고, 로그인 성공과 실패가 구분되도록 구현했습니다.
로그인 상태는 Streamlit의 session_state를 이용해 유지되도록 만들었습니다.

추천 시연
한 번 틀린 비밀번호 입력 → 실패 메시지
다시 맞는 비밀번호 입력 → 성공

이렇게 하면 더 좋아 보임.

4. 직접 검사 시연
화면

검사 시작하기 클릭
문항 몇 개 직접 선택
진행바랑 섹션 설명 보이게

대본

먼저 직접 검사 기능입니다.
이 앱은 총 6개 섹션, 60문항으로 구성되어 있고, 각 문항은 4지선다의 상황형 질문으로 이루어져 있습니다.
사용자는 가장 끌리는 선택지를 고르게 되고, 각 선택지는 세계관, 역할군, 태그, 그리고 빅파이브 보정 점수에 연결됩니다.

여기서 팁

60문항 다 찍을 필요 없음.
앞에서 2~3문항 정도만 직접 보여주고 바로 결과 화면으로 넘어가도 된다.

방법 1

미리 거의 다 풀어놓고 마지막 몇 문제만 찍기

방법 2

설명하면서 “이후 모든 문항에 응답하면 결과를 확인할 수 있습니다”라고 말하고 결과 화면으로 넘어가기

실전적으로는 미리 거의 다 풀어둔 상태가 제일 편하다.

대본

모든 문항에 응답하면 이렇게 결과 리포트가 생성됩니다.
결과는 하나의 세계관, 하나의 역할군, 두 개의 태그를 조합해서 출력되며,
그에 맞는 프롤로그와 진행 조언도 함께 생성되도록 만들었습니다.

아래에는 Plotly를 활용해서 세계관 점수, 역할군 점수, 상위 태그 점수를 시각화했습니다.
이 부분을 통해 사용자의 선택이 어떤 방향으로 누적되었는지 한눈에 볼 수 있도록 했습니다.

6. 랜덤 선택 결과 보기 시연
화면

랜덤 선택으로 결과 보기 클릭
결과 페이지 뜨는 것 보여주기

대본

다음은 랜덤 선택 기능입니다.
이 기능은 결과만 무작위로 보여주는 것이 아니라, 60개 문항의 선택지를 실제로 랜덤 선택한 뒤 같은 점수 계산 로직을 적용해서 결과를 생성합니다.
그래서 데모나 빠른 테스트용으로 사용할 수 있습니다.

포인트

이 설명 괜찮다. 교수님이 보면 “아, 그냥 fake 결과가 아니구나” 하고 이해하기 좋다.

7. 캐릭터 아카이브 시연
화면

캐릭터 아카이브 들어가기

결과 프리뷰 보여주기

대본

다음은 캐릭터 아카이브 기능입니다.
여기서는 실제 점수 계산 없이, 사용자가 세계관과 역할군, 태그 조합을 직접 선택해서 결과를 미리 확인할 수 있습니다.
이를 통해 다양한 캐릭터 조합을 살펴볼 수 있도록 했습니다.

여기서 추천 시연

예를 들어:

세계관: 사이버펑크
역할군: 기술자
태그: 탐구형 / 공략형

이런 식으로 골라서 보여주면 예쁘다.

8. 캐싱 설명
화면

코드 화면까지 꼭 보여줄 필요는 없고, 앱 화면에서 말만 해도 됨
가능하면 VS Code에서 @st.cache_data 잠깐 보여줘도 좋음

대본

이 프로젝트에서는 Streamlit의 st.cache_data를 사용해 JSON 데이터를 캐싱했습니다.
질문 데이터, 사용자 데이터, 세계관 데이터, 역할군 데이터, 결과 텍스트 데이터를 반복해서 다시 읽지 않도록 해서 실행을 더 효율적으로 만들었습니다.
즉, 새로고침이나 다시 실행이 일어나더라도 같은 데이터를 불필요하게 반복 로딩하지 않도록 구성했습니다.

더 짧게 말하고 싶으면

캐싱 기능은 st.cache_data를 이용해 JSON 파일 로딩에 적용했습니다.
질문 데이터와 결과 텍스트 데이터를 반복해서 읽지 않도록 해서 앱 실행 효율을 높였습니다.
